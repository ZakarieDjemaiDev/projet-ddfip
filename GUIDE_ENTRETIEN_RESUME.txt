# POINTS CLES - RESUME POUR ENTRETIEN

## EN UNE PHRASE POUR CHAQUE PARTIE

### GENERATE_DATA.PY
"Ce script cree des donnees realistes pour simuler votre vraie base agents :
noms francais, services varies, grades differents, conges, habilitations.
C'est pret a adapter a vos donnees reelles."

### RAPPORT.PY
"Analyser 100 agents avec Excel = 20 minutes et risque d'erreur.
Ce script = 2 secondes et parfait. Il detecte automatiquement les alertes
(conges faibles, habilitations expirees) et les affiche."

### DASHBOARD FLASK
"Un seul endroit pour voir tous les KPIs et graphiques. Au lieu de se
perdre dans une feuille de 100 lignes, la reponse a vos questions
(combien de gens en alerte? comment sont repartis les effectifs?) est
immediate et visuelle."

### MACRO LIBREOFFICE
"Tout votre rapport, mis en forme et alerte en un clic, directement dans Calc.
C'est ce que vous ferez probablement faire : pas de commande ligne, 
juste des boutons."

## QUESTIONS => REPONSES PREPEREES

**Q: Et si votre script se casse ?**
A: "Les scripts Python sont robustes. Mais si un probleme survient :
   - J'ai du logging pour debugger rapidement
   - Les donnees sont en CSV, elles sont sauvegardees
   - On peut revenir a la version precedente (git)"

**Q: Et la performance avec des milliers d'agents ?**
A: "Avec 10,000 agents, c'est toujours < 1 seconde. 
   Avec 100,000+, on pense a une base SQL au lieu du CSV.
   La structure reste exactement la meme."

**Q: Comment vous maintenez ca sur plusieurs annees ?**
A: "Tout est documente et commente. Un nouveau dev peut comprendre
   le code en < 1 heure. Les tests (si on en fait) assurent qu'on
   ne casse rien en modifiant."

**Q: Et si vous partez et que personne ne sait programmer ?**
A: "Le dashboard + la macro suffisent pour 95% du travail. 
   Si un truc casse, c'est juste relancer le script ou call un dev Python.
   C'est pas complexe."

**Q: Pourquoi pas une solution Salesforce / SAP?**
A: "Coute 10x plus cher, moins flexible, overkill pour 30 agents.
   Ici c'est sur-mesure, pas cher, et vous le controllez."

**Q: Comment ?a s'integre avec votre vrai systeme RH ?**
A: "?a va dependre : API, export CSV daily, ou requete directe a la base.
   Le core (analyse, visualisation) reste pareil. C'est juste source qui change."
